The three observations that warranted a fix were all a11y holes in the actually-rendered components: (1) `<ProgressBar>` renders as a bare `<div class="bar"><span/></div>` with no `role="progressbar"` / `aria-valuenow` — the Today-card progress bar and per-assignment row bars are invisible to assistive tech; (2) the Active/Completed `<Segmented>` container reports `role: null, ariaLabel: null` — same pattern as the trainer-studies-list fix in PR #75; (3) the `MiniStat` tiles ("0 active" / "0 completed") are bare nested divs with no grouping, so screen readers read the value and label as two unrelated adjacent nodes. Lower-priority observations not addressed in this PR: the 11px `--text-faint` label colour against `--inset-bg` is tight on contrast at small sizes, but the swatch is global and out of scope for a single-feature rotation.

Fixed all three with purely-additive a11y: `src/components/ui/atoms.tsx` `ProgressBar` now carries `role="progressbar"` + `aria-valuemin/max/now` + `aria-valuetext` + optional `aria-label` (one new prop); `src/student/DashboardPage.tsx` passes `ariaLabel="Filter assignments by state"` to the filter Segmented, `ariaLabel="Today's average progress across assignments"` to the Today bar, `ariaLabel={`${a.name} progress`}` to each row bar, and adds `role="group"` + `aria-label={`${value} ${label}`}` on `MiniStat`'s outer div (with the inner spans `aria-hidden="true"` to avoid double announcement). No copy / layout / visual change.
