Clicking "Import from Lichess" on /trainer/studies opens the
NewOpeningStudyDialog with `lichessHint=true` — title reads "Import from
Lichess", the hint paragraph explains the next screen will ask for a
PGN, and the CTA reads "Create + import PGN →". A11y, focus styles, and
Escape-to-close all behave correctly.

What the 375x812 mobile pass exposed: the dialog renders at **900px
tall** inside an **812px viewport**. The `modal-backdrop` uses
`align-items: center`, so the excess height is split top and bottom and
clipped — on a real phone the header (title + Close button) and the
form footer (Cancel / Create + import PGN) end up partially or fully
outside the visible area, with no way to scroll to them because the
dialog has no max-height/overflow and the body scroll is locked.

Smallest fix in `src/components/ui/Dialog.tsx`: cap the dialog inner box
at `calc(100dvh - 32px)` and set `overflowY: auto`. On short screens the
dialog now stays inside the viewport and its own content scrolls; on
desktop and tall-mobile screens it is a no-op. This benefits every
dialog using this component (new study / invite student / import
Lichess / API key creation / etc.) — they were all subject to the same
clip when their natural height exceeded the viewport.
