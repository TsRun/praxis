The spec surfaced a recognizable batch of UI/a11y issues, but every actionable one is already addressed by open PR #78 (`landing-page-render`), which modifies the same `src/auth/SignInUpForm.tsx` component this feature exercises:

- `MODE TOGGLE: [{ text: "Sign in", ariaPressed: null }, { text: "Create account", ariaPressed: null }]` — the two mode tabs expose no `aria-pressed` / `role` state. PR #78 adds `role="group"` + `aria-label="Account mode"` on the wrapper and `aria-pressed={mode === 'in' | 'up'}` on each tab.
- `NAME INPUT a11y: { id: null, labelsFor: 0, wrappingLabel: null, type: "text" }`, `EMAIL INPUT a11y: { id: null, labelsFor: 0, type: "text" }`, `PASSWORD INPUT a11y: { id: null, labelsFor: 0, wrappingLabel: null }` — the three inputs are not programmatically labelled (no `htmlFor`, no `id`, no wrapping `<label>`). PR #78 wires `useId()` ids and `<label htmlFor>` on all three, and upgrades the email input from `type="text"` to `type="email"`.
- `ROLE GRID container: { role: null, ariaLabel: null, ariaLabelledby: null }` and `ROLE PICKER BUTTONS` reporting `{ text: "TrainerI coach others", ariaPressed: null, ariaLabel: null }` (etc. for Student / Solo) — the role picker has no group semantics, no toggle state, and screen readers run the title + sub together as one word. PR #78 converts the "I am a" orphan label to a `<div id={rolesGroupId}>`, wraps the buttons in `role="group" aria-labelledby={rolesGroupId}`, and gives each button `aria-pressed={on}` plus `aria-label={`${title} — ${sub}`}`.
- `ROLE-EMPTY ERROR a11y: { text: "Pick at least one role.", role: null, ariaLive: null }` — submitting Create-account with zero roles selected shows the danger message but the region is conditionally mounted and has no live-region semantics. PR #78 switches the error region to always-rendered `role="alert" aria-live="polite"`.
- `MOBILE OVERFLOW: { scroll: 382, client: 375 }` — the hero glow `-10%` inset escapes the body on 375-wide viewports and produces a 7px horizontal scroll. PR #78 adds `body { overflow-x: clip; }` in `src/index.css`.
