UI quality issue caught: menu items have **no visible hover background**
(getComputedStyle reported `rgba(0, 0, 0, 0)` both at rest and on hover).
Root cause — the `<button class="dropdown-menuitem">` had an inline
`style={{ background: 'transparent' }}` that overrode
`.dropdown-menuitem:hover { background: var(--chip-bg) }` from
src/index.css (inline styles outrank class selectors). The same override
silently broke the `.dropdown-menuitem:focus-visible` background that
keyboard users rely on to track focus.

Fix: dropped `background: 'transparent'` from the inline style on
NewStudyMenuItem (src/trainer/StudiesPage.tsx) and moved the transparent
default into the `.dropdown-menuitem` CSS rule (src/index.css). Class-based
hover and focus-visible backgrounds now win. Typecheck + npm test pass.
No page errors, no application console errors during the prod run.
